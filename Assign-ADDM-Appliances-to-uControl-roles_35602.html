uControl Documentation 5.0 : Assign ADDM Appliances to uControl roles

Created by Steve Hicks on Jul 08, 2016

 

Configure Roles

Once ADDM appliances have been added to uControl they can be assigned roles which controls how uControl interacts with the Appliance. For example if you have a DEV appliance which you use for testing TPL patterns before you deploy to a production appliance then you can assign a DEV TPL appliance and a PROD TPL appliance in uControl. This will result in all TPL being deployed to your DEV appliance and once tested deployed to your PROD appliances.

  1. Login to uControl as an Admin privileged user and select the uAdmin module
  2. Under the uManage TAB click on the 'Manage Appliance Roles' TAB.



  3. Depending on the role that you wish uControl to use the specific ADDM appliance for - click the Add Appliance button and select the required ADDM appliance from the list.

 

 

Role Definitions

Role NameDescriptionRequired
uMap TPL ProductionSigned off TPL patterns will be uploaded and activated to appliances in this role
uMap TPL TestIn testing TPL patterns will be uploaded and activated to appliances in this role 
uChange Visualisations ProductionVisualisations will be gathered for all signed off applications from the appliance in this role 
uChange Visualisations TestVisualisations will be gathered for all in progress applications from the appliance in this role 
uManage Production ScanneruManage graphs will be built for scanning appliances in this role 
uManage Production ConsolidationuManage graphs will be built for consolidation appliances in this role 
Subnet ScanneruManage will use all appliance listed in here for all Subnet Scanning tasks configured 
uManage Check LoginuManage will use all appliance listed in here for all login checks requested by users
