uControl Documentation 5.0 : Users and Groups

Created by Steve Hicks on Jul 08, 2016

uControl support standalone and SSO configurations for Users.

Create Standalone User

  1. Login to uControl as an administrator privileged account
  2. Navigate to uAdmin -> System -> Manage User

  3. Click the New User button



  4. Enter the account details and select the groups that the account should be a member of. 

Configure SSO

Setting up uControl:

uControl typically sits behind an Apache reverse proxy taking requests to /uControl on the http port 80 and forwarding them to uControl which sits on port 8080 on url /uControl.

If a site uses https then Apache can be configured with an SSL Certificate and configured to accept uControl requests on the https port (443) instead and forward them to port 8080.

uControl typically runs under a Linux operating system and most distributions of Linux come pre installed with “iptables”. IPTables is a command-line firewall utility that uses policy chains to allow or block traffic. When a connection tries to establish itself on your system, IPTables looks for a rule in its list to match it to. If it doesn’t find one, it resorts to the default action. You can configure IPTables to block non-local access to port 8080, which would block all access to uControl that has not been forwarded by the local Apache reverse proxy.

You can either add an SSO agent to the Apache reverse proxy or you can sit the uControl install behind a Secure Proxy Server such as “SiteMinder Secure Proxy Server” SPS. The SiteMinder Secure Proxy Server (SPS) is a stand-alone server that provides a proxy-based solution for access control. The SPS employs a powerful proxy engine that provides a network gateway for the enterprise and supports multiple session schemes that do not rely on traditional cookie-based technology. This guide discusses how to configure and deploy the SPS in an enterprise.

 

Configuring uControl:

There are only two parameters needed to configure uControl, these can be configured under uAdmin -> System -> System Params

SSOHeaderName - the name of the token SSO puts in the http request header containing the username. (A blank value means SSO has not been configured.)

Logout URL – where the user is redirected after they have logged out of uControl.

If a SSOHeaderName has been specified and uControl cannot find the header, uControl will dump all the http headers it can see in the system logs.


User Groups

The groups control which areas / modules of uControl the user has access to.

GroupDescription
AdministrationAccess to all modules in uControl with Admin privileges
ModellerAccess to uMap and uChange
ReportingAccess to uReport
ManagementAccess to uManage

Groups can be combined to enable users to have access to multiple modules. As an example if you wanted a user to be able to model applications and view / use reports then you would add their account to the Modeller and Reporting group.
