uControl Documentation 5.0 : uMap Application Data

Created by Steve Hicks on Jul 08, 2016

uMap requires information on hosts, products and processes to enable to application modelling functionality in uControl. We also need to setup the schedule of the TPL deployments to you production appliances (As defined in the uControl roles section).

Configure uMap Data Import Schedule

  1. Navigate to uAdmin -> uMap -> Manage Imports
  2. Under the 'Imported run time Config' section of the screen ensure the 'Import Enabled' and 'Download Enabled' are both set to true for all 4 imports
  3. Setup the Time and days of the week the importer should run at

Be sure to scheduled these imports are the correct time. It's advised that these imports are schedules post ADDM scanning to ensure the data used for application modelling in uMap is up to date.

 

Run Adhoc Data Import

  1. Navigate to uAdmin -> uMap -> Manage Imports
  2. Under the 'Last Download Status' table select the 'Run Now' button to run the importers.

Check Last Import Status

  1. Navigate to uAdmin -> uMap -> Manage Imports
  2. Under the 'Last Download Status' section of the screen will be the last status.

Debug Issues With Last Import

  1. Navigate to uAdmin -> uMap -> Manage Imports
  2. Under the 'Last Download Status' notification section is a drop down off the appliances configured as per the uControl TPL roles.
  3. Select the appliances one by one from the drop down, the table will show the 'File Downloaded' and 'File Imported' status.
