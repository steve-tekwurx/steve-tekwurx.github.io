uControl Documentation 5.0 : Ignoring Alerts

Created by Steve Hicks on Jul 08, 2016

Why

If the change alert contains information / identifiers that does not belong to it's represented application then the alert will need to be ignored.

This meaning that the identifier will be added as an exclusion that will suppress this and any future alerts that meet this criteria. The criteria being host + identifier.

Where

The following alert types support the ignore function:

  • Possible Additional Components
  • Possible Additional Processes

To Ignore an Alert

  1. Navigate to uChange -> New
  2. Drop down the filter to show either the possible additional production or process alert types
  3. Select the alert and select ignore from the popup or check multiple alerts and use the button provided

If you are logged in an an administrator of uControl additional filtering by 'User' is available - this is the user that was assigned / completed the application modelling process in uMap for the application
