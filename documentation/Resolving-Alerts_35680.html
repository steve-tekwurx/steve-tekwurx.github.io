uControl Documentation 5.0 : Resolving Alerts

Created by Steve Hicks on Jul 08, 2016

Why

 

Possible Additional Component / Process Alert Types

If the change alert contains information / identifiers that does belong to it's represented application then the alert will need to be resolved.

This means that the identifier will be linked to the application and the model flagged to redeploy to the ADDM production TPL appliances configured in uAdmin -> uManage -> Appliance Roles

uChange Additional Alert Type

If the change alert contains Hosts that have been decommissioned from an application then the alert will need to be resolved.

This means that the host will be removed from the linked hosts for the application and the model flagged to redeploy to the ADDM production TPL appliances configured in the uAdmin -> uManage -> Application Roles

This redeploy will only be sent to ADDM as per the uMap Production TPL schedule defined in uAdmin -> uMap -> Manage uMaps

 

Where

The following alert types support the resolve function:

  • Possible Additional Components
  • Possible Additional Processes
  • uChange Additional

To Resolve an Alert

  1. Navigate to uChange -> New
  2. Drop down the filter to show either the possible additional production or process alert types or uChange Additional
  3. Select the alert and select resolve from the popup or check multiple alerts and use the button provided

If you are logged in an an administrator of uControl additional filtering by 'User' is available - this is the user that was assigned / completed the application modelling process in uMap for the application
