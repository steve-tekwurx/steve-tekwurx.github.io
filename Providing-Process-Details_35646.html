uControl Documentation 5.0 : Providing Process Details

Created by Steve Hicks on Jul 08, 2016

Step 4 enables you to identify, select and match you application processes. This step is split into 2 ways of completing this.


Match by Regex (Advanced Users)

If you are experienced with regex and are familiar with your application processes you can simply enter regex expressions to match your processes by Username, Command and Arguments.

  1. Enter the key information (Regex) into the User, Command or Arguments fields and select the 'submit regex' button

  2. Once you have submitted the key information, the results are show in the 'Patterns that identify linked processes' grid.
    1. Linked Hosts - This shows the total hosts that are linked to the application that are matched by this regex
    2. Extra Hosts - This shows the total hosts that are matched by this regex

 

Select Process and Match

  1. Select the 'Select By Process' button and the top left side of the screen

  2. Enter the key information that you wish to search for in the first row of the displayed grid to filter the processes and select a process from the resulting rows

    This view will always default to showing all processes running on all hosts discovered / imported into uControl.

    If you would like to set a filter to the list to only show processes running on the selected / linked hosts supplied in the Host step, select the checkbox for 'Processes filtered by hosts linked to this application'

     

    This grid shows 2 columns to help you understand if the processes is unique to you application or generic across your estate. These are:

    • Linked Hosts - This shows the total hosts that are linked to the application that are matched by this regex
    • Extra Hosts - This shows the total hosts that are matched by this regex

  3. A popup will be displayed showing details of the selected process. Selected the 'OK' button to link this process to the application.

    Although the process has been selected, at this point it has not been matched by plain text or regex characters.

  4. To supply the matching criteria for a process, select a linked process from the 'Processes linked to this application' grid
  5. An 'Update selected process' popup will be displayed. Here we can enter the matching criteria for the process. Follow the below section for 'Creating Process Identifiers'

 

 

Creating Process Identifiers

This matching section enables you to use plain text or regex characters for advanced pattern matching.

Enter the information (user, command, arguments, Name) and click submit regex.

Note that only the Name is mandatory

 

When you click to submit uControl checks the regex against the processes you selected and also all the processes discovered by ADDM and displays whether you will cause a match against any hosts not included in Step 2 (link hosts).

Once a regex has been created and successfully matched against the process – the process table is updated to show the regex ID that is used to match it.



Click the running count of matched, unmatched and extra hosts to get a list of what hosts are now linked to your application and what has caused them to be matched.

When you click Next Step – uMap will alert you to there being extra hosts in the model. At this time you can either cancel and go back and modify the regex or click “Link” which will add the new hosts to your application.
